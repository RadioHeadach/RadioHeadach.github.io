When I was learning SwiftUI from Appleâ€™s tutorial, I found a bug.

Apple Developer Documentation

https://developer.apple.com/tutorials/sample-apps/editinggrids

This app is a simple, contains two view: My Symbols View and Symbol Picker View. Tapping the â€˜plusâ€™ icon from the navigation bar will present the picker view, when you tap any icon in the picker view, the icon you selected will be added to your collection.

The Bug

The problem is,

  1. the collection to hold the selected icon is a mutable array, which accepts the iconâ€™s name. Inside the LazyVGrid , it uses for each method to iterate the selected array, using array itself as id: id: \.self, which means you canâ€™t hold to identical icon name inside the array
    ForEach<Array<String>, String, ModifiedContent<NavigationLink<ModifiedContent<ZStack<TupleView<(ModifiedContent<ModifiedContent<ModifiedContent<ModifiedContent<ModifiedContent<Image, _AspectRatioLayout>, _EnvironmentKeyWritingModifier<Optional<SymbolRenderingMode>>>, _EnvironmentKeyWritingModifier<Optional<Color>>>, _SafeAreaRegionsIgnoringLayout>, _ClipEffect<RoundedRectangle>>, Optional<ModifiedContent<Button<ModifiedContent<ModifiedContent<ModifiedContent<Image, _EnvironmentKeyWritingModifier<Optional<Font>>>, _EnvironmentKeyWritingModifier<Optional<SymbolRenderingMode>>>, _ForegroundStyleModifier2<Color, Color>>>, _OffsetEffect>>)>>, _PaddingLayout>, ItemDetail>, ButtonStyleModifier<PlainButtonStyle>>>: the ID house.fill occurs multiple times within the collection, this will give undefined results!
LazyVGridLayout: the ID house.fill is used by multiple child views, this will give undefined results!
  2. after you added an empty icon into the collection, you can still present the picker view, but when you add nothing but just pulling down to make the view dismiss, guess what, you will add an empty icon AGAIN!

Why it happened?

Letâ€™s dive into the code:

As we can see from above, SymbolGrid uses a binding object to hold the selected iconâ€™s name, after picking the icon, the param @State private var selectedSymbolName: String? will be modified and change to the name you picked. selectedSymbolName will be empty when initialized, but when you pick an icon for the first time, it will automatically changed to the latest iconâ€™s name that you pick, until you pick another one. So when you pick an icon once, then you pull down and not picking any icon for the second time, the selectedSymbolName will still be the same, and when dismissed, function addSymbom() is call and an empty icon will be added.
