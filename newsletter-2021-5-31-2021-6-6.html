如何为你的 iOS app 接入 keychain

最近开始做新项目，在测评产品原型的时候，发现同事在登陆界面放了两个按钮，“记住账号”和“记住密码”。我才发现好久没有在手机app上见过这两个按钮了，现在都是用keychain来生成和记住密码的。

于是我调研了一下怎么让app接入keychain：

Apple Developer Documentation

https://developer.apple.com/documentation/security/password_autofill?language=objc

Apple Developer Documentation

https://developer.apple.com/documentation/security/password_autofill/enabling_password_autofill_on_a_text_input_view?language=objc

Apple Developer Documentation

https://developer.apple.com/documentation/security/password_autofill/enabling_password_autofill_on_an_html_input_element?language=objc

按照苹果的规范配置associated domain，在textfield和html标签规定好属性，就能同时在app端跟网页端让系统弹出password autofill的选项。

KVO 和 NSNotificationCenter 的区别

What are the correct uses and differences of KVO and NSNotificationCenter?

KVO is specifically for the change of properties. NSNotificationCenter can be used in a much broader context for all kinds of notifications, not only those dealing with property value changes. Plus, you can create your own notifications to send out to any other interested object that formerly subscribed to this notification as an observer.

https://stackoverflow.com/questions/26782029/what-are-the-correct-uses-and-differences-of-kvo-and-nsnotificationcenter

总而言之，KVO是拿到类属性的值和其变化，继承自NSObject。而NSNotificationCenter不仅提供属性的变化，还能自定义通知发送方和接收方。这么看来NSNotificationCenter的业务范围更广呢。
